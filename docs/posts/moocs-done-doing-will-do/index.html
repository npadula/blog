MOOCs: Done, Doing, Will Do

June 12, 2020 3 minutes
  • moocs
  • education
  • courses

A list of online courses that I did, am doing, or would like to do sometime. Courses listed here can follow a structured/formal approach (they can have excercises like courses on Pluralsight and Coursera) or just be a simple collection of Youtube lectures.

Frontend

React

  • (Done) Corey House’s React/Redux on Pluralsight. Very useful for understanding the Flux architecture and how Redux implements it.
  • (Done) Daniel Stern’s Redux-Saga. Good complement to the Redux course, also a good resource to understand generators in ES6.

Angular

  • (Done) Angular Fundamentals by Jim Cooper. Good for an overall picture of how Angular works. Longer than usual but that is to be expected because Angular itself it’s a broader framework than React, Vue, and others. Very comprehensive, you get a look at all the main constructs in Angular.

Machine Learning

  • (Dropped out) Andrew Ng’s Machine Learning (Coursera). Pretty good, intuitive course, I dropped out after the neural networks unit because it felt a little bit shallow, and I wanted some more details on the underlying math.
  • (Done) Yaser Abu-Mostafa’s Learning From Data. One of the best courses I’ve ever watched both online and IRL. The lectures are engaging and provide tremendous insight on the fundamental learning problems. Highly recommended.
  • (Dropped out) Stanford CS229. Started this one after Lelarning From Data. Since it is Stanford I thought it would be a more deep version of Andrew Ng’s Coursera MOOC, but I was severely dissappointed. I feel not much math insight is gained from the lectures, and classes are kind of boring. The lecture notes are handy though.
  • NYU Deep Learning. Focused on Deep Learning, so it’s more narrow than CS229. Seems to have a lot practical lessons so that is cool.
  • fast.ai has a lot of stuff, more practiced oriented.

Software Architecture / System Design

  • (Done) Pluralsight’s Domain-Driven Design Path.. Amazing series of courses, it covers many aspects like Clean Architecture, theoretical DDD concepts, implementation of DDD to application design and refactoring, combination with CQRS and Event Sourcing, and some other patterns. Recently they added Vladimir Khorikov’s latest EF Core course, which seems really promising too.

Distributed Systems

There are many in this category that I want to do:

  • MIT 6.824 is entirely available for free, with auto-grading scripts and everything!
  • Lambda Architecture seems to have a pretty good overview of a widely used big data architecture.
  • (Doing, ~80%) Scaling Applications with Microservices, MassTransit and RabbitMQ is more about queues and events than anything else, but the last units do seem to provide explanations of common patterns.
  • Microservices Architectural Design Patterns Playbook seems to be more of a reference than anything, but it may be worth a shot.

DevOps

  • There are several k8s courses that I want do so to familiarize myself with some internal aspects. Pluralsight have two paths, the developer path content seems interesting as an overview, but I’m also interested on networking and monitoring, which is covered in the administration path.

Functional Programming

  • (Done) Applying Functional Principles in C#. Good intro to useful patterns in C#.

  • Functional Programming in Scala. Taught by the Scala creator himself. Appears to have several practical application lectures. Seems really promising.

Blockchain

  • (Done) Ivan Mushketyk’s Ethereum Development course. It’s a very good course because it provides a good, broad picture of how blockchain architectures work, as well as a more in depth look of how to implement DApps with Solidity.

Mathematics

  • Measure Theory. No reason for this, just looks pretty cool.
  • Category Theory. Kind of scared of this one, been a while since I worked in stuff so abstract.
